# GeekeriOS

**An AI-native RevOps operations platform that turns Claude Desktop into a multi-agent command center with full observability and persistent memory.**

GeekeriOS connects [VoltAgent](https://github.com/VoltAgent/voltagent) (agent orchestration), [ruflo](https://github.com/ruvnet/ruflo) (cross-session vector memory), and [VoltOps](https://console.voltagent.dev) (real-time tracing) through the [Model Context Protocol](https://modelcontextprotocol.io). It runs entirely on your local machine — your API keys, your data, your disk.

```
Claude Desktop Chat ←stdio→ VoltAgent MCP Server (:3141)
                                    │
                  ┌─────────────────┼─────────────────┐
                  │                 │                  │
           Supervisor Agent    ruflo MCP         VoltOps Console
         (revops-orchestrator) (memory tools)   (console.voltagent.dev)
                  │
    ┌─────────────┼─────────────┬─────────────┐
    │             │             │             │
CRM Analyst  Data Researcher  Report Writer  Strategy Advisor
```

You chat in Claude Desktop. The supervisor breaks complex requests into subtasks and delegates to specialist sub-agents (real Claude instances). ruflo stores patterns and findings that persist across sessions. VoltOps traces every delegation, tool call, and LLM response with timing and token costs.

Everything is config-driven. Two JSON files define all agents, skills, and tools — editable from Claude Desktop's Chat tab using natural language.

---

## What It Does

- **Multi-agent orchestration** — A supervisor agent delegates to 4 specialists working in parallel. Pipeline audits, revenue forecasts, and client deliverables are generated by the right expert for each subtask.
- **Persistent memory** — ruflo's vector memory stores consulting frameworks, client findings, and learned patterns across sessions. `memory_search` retrieves relevant past work automatically.
- **Full observability** — VoltOps Console shows every agent delegation, tool call, LLM request/response, token usage, and latency in a visual execution trace. No black box.
- **Config-driven** — Add agents, update instructions, assign skills, and toggle tools by editing JSON files. No code changes for 90% of modifications.
- **Local-first** — Runs on your machine. Data stays on your disk. Traces persist to local SQLite.

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/geekeriOS.git
cd geekeriOS
npm install
```

### 2. Configure your API key

```bash
cp .env.example .env
```

Edit `.env` and add your Anthropic API key:

```
ANTHROPIC_API_KEY=sk-ant-api03-your-key-here
```

### 3. Connect Claude Desktop

Open your Claude Desktop config file:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Replace or add the following (update paths to match where you cloned the repo):

```json
{
  "mcpServers": {
    "revops": {
      "command": "bash",
      "args": [
        "-c",
        "export PATH=/PATH/TO/NODE20/bin:$PATH; cd /PATH/TO/geekeriOS; npx -y ruflo@latest init >/dev/null 2>&1 || true; npx -y ruflo@latest daemon start >/dev/null 2>&1 || true; exec npx tsx src/index.ts"
      ]
    }
  }
}
```

**macOS with nvm example:**

```json
{
  "mcpServers": {
    "revops": {
      "command": "bash",
      "args": [
        "-c",
        "export PATH=$HOME/.nvm/versions/node/v20.19.6/bin:$PATH; cd $HOME/Desktop/geekeriOS; npx -y ruflo@latest init >/dev/null 2>&1 || true; npx -y ruflo@latest daemon start >/dev/null 2>&1 || true; exec npx tsx src/index.ts"
      ]
    }
  }
}
```

### 4. Restart Claude Desktop

Fully quit (Cmd+Q on macOS, not just close the window) and relaunch. The MCP server starts automatically.

### 5. Test it

In the Claude Desktop **Chat tab**, try:

> Run a pipeline audit for a B2B SaaS company with $2M ARR. They have 45 deals across 5 stages, average sales cycle of 60 days. Demo-to-proposal conversion is 40%, proposal-to-negotiation is 55%, negotiation-to-close is 35%. Identify the biggest bottleneck and recommend fixes.

You should see the orchestrator delegate to the CRM Analyst and Data Researcher, then pass results to the Report Writer.

### 6. View traces

Open **https://console.voltagent.dev** — it auto-connects to `localhost:3141` and displays the full execution trace.

## Configuration

### agents.json — Agent Definitions

Defines the supervisor and all sub-agents. Each agent has:

| Field | Purpose |
|---|---|
| `name` | Display name, used for delegation routing and VoltOps display |
| `purpose` | Short description the supervisor sees when choosing who to delegate to |
| `instructions` | Full prompt the agent uses when executing a task |
| `model` | Anthropic model string (e.g. `claude-sonnet-4-20250514`) |

The `supervisor` section has an additional `guidelines` array — rules injected into the supervisor's system message.

**Default agents:** CRM Analyst, Data Researcher, Report Writer, Strategy Advisor

### skills.json — Skills and Tools

**Skills** are methodology modules injected into agent instructions at startup:

| Field | Purpose |
|---|---|
| `id` | Unique identifier |
| `name` | Human-readable label |
| `appliesTo` | Array of agent names that receive this skill |
| `content` | Instruction text appended to the agent's prompt |

**Tools** are executable functions toggled per agent:

| Field | Purpose |
|---|---|
| `id` | Must match a key in the `builtInTools` map in `src/index.ts` |
| `appliesTo` | Which agents get access |
| `enabled` | true/false toggle |

**Default skills:** Pipeline Audit Methodology, Revenue Forecasting Model, Tech Stack Assessment, Client Deliverable Standards, CRM Data Quality Checklist

**Built-in tools:** `calculate_metrics` (11 RevOps formulas), `format_currency` (K/M/B notation)

## Modifying the System

The fastest way to modify GeekeriOS is through Claude Desktop Chat, since it has filesystem access.

### Add an agent

```
Read agents.json and add a new sub-agent called "Customer Success Analyst"
that specializes in churn analysis, NRR tracking, and customer health
scoring. Use model claude-sonnet-4-20250514. Save the file.
```

### Add a skill

```
Read skills.json and add a skill called "MEDDPICC Qualification" that
applies to CRM Analyst and Strategy Advisor. Content should cover Metrics,
Economic Buyer, Decision Criteria, Decision Process, Identify Pain,
Champion, Competition. Save.
```

### Enable a tool for an agent

```
Read skills.json and enable the calculate_metrics tool for Strategy Advisor. Save.
```

### Update the supervisor

```
Read agents.json and add a new guideline to the supervisor:
"Always check data quality before running pipeline analysis". Save.
```

### Review current configuration

```
Read both agents.json and skills.json, summarize all agents with
their assigned skills and enabled tools.
```

**After any edit:** Cmd+Q Claude Desktop and relaunch to apply changes.

### Adding new executable tools

New tools with custom logic require editing `src/index.ts`:

1. Add the tool to the `builtInTools` map using `createTool` with a Zod schema
2. Add a matching entry in `skills.json` with `enabled: true` and the `appliesTo` array

## ruflo — Memory Layer

ruflo provides cross-session vector memory via MCP tools. Based on extensive testing, these tools have verified working backends:

| Tool | What it does |
|---|---|
| `memory_store` | Save key-value data to persistent sql.js storage |
| `memory_search` | Vector search across stored patterns |
| `memory_retrieve` | Fetch a specific key |
| `memory_list` | List stored keys in a namespace |
| `hooks_session-start` | Begin a tracked session |
| `hooks_session-end` | End a session |
| `swarm_health` | Check coordinator status |
| `coordination_metrics` | Latency/throughput stats |

**Note:** ruflo's agent execution tools (`agent_spawn`, `task_create`, `hive-mind_init`, `workflow_run`) create metadata records but have no execution runtime in the MCP context. VoltAgent's native supervisor + sub-agent system handles actual agent orchestration. ruflo is used solely for its functional memory and session management.

## VoltOps Observability

Open **https://console.voltagent.dev** while the platform is running. It connects to `localhost:3141` via WebSocket.

**What gets traced:**
- Every `delegate_task` call from supervisor → sub-agent
- Every sub-agent's LLM request and response
- Every ruflo memory tool call
- Token usage per request
- Response latency per step
- Full execution tree

Traces persist to `.voltagent/observability.db` (LibSQL/SQLite) and survive process restarts.

For remote persistence, add keys to `.env`:

```
VOLTAGENT_PUBLIC_KEY=pk_...
VOLTAGENT_SECRET_KEY=sk_...
```

Get keys at [console.voltagent.dev/tracing-setup](https://console.voltagent.dev/tracing-setup).

## Troubleshooting

| Issue | Fix |
|---|---|
| MCP server not appearing in Claude Desktop | Fully quit (Cmd+Q) and relaunch. Check `~/Library/Logs/Claude/` for MCP server logs. |
| `EBADENGINE` warnings about Node version | Claude Desktop defaults to Node 18. Ensure the startup chain includes `export PATH=/path/to/node20/bin:$PATH` |
| stdout pollution errors (`Unexpected token`) | All setup commands before `mcp start` must redirect both stdout and stderr: `>/dev/null 2>&1` |
| Shell script "Operation not permitted" | macOS Gatekeeper blocks external scripts. Use inline `bash -c` commands instead. |
| Port 3141 already in use | Check with `lsof -i :3141`. Kill the existing process or change the port in `src/index.ts`. |
| ruflo `agent_spawn` / `task_create` fails | These tools create phantom records with no execution runtime. Use VoltAgent sub-agents for actual orchestration. |
| VoltOps Console shows "disconnected" | Verify the dev server is running: `curl http://localhost:3141/health` |

## Known Limitations

- **Cowork tab** does not support local stdio MCP servers ([Anthropic bug #20377](https://github.com/anthropics/claude-code/issues/20377)). Use the Chat tab.
- **Custom connectors** in Claude Desktop require HTTPS + OAuth. Local HTTP servers cannot be added as connectors.
- **Config changes require restart.** `agents.json` and `skills.json` are read at startup.
- **ruflo daemon instability.** The daemon can report "running" while actually stopped ([ruflo #1283](https://github.com/ruvnet/ruflo/issues/1283)). Use `hooks_session-start` as the reliable initialization path.
